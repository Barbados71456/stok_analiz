# Сток-анализ (stok_analiz)

Следит за гугл-таблицей стока машин, собирает историю изменений, оценивает
рыночную стоимость каждой машины и выдаёт:

- **покупателю** — вердикт «брать / держать / мимо» с учётом скидки к рынку и
  юридических запретов (запрет суда / ФССП = машину нельзя перерегистрировать);
- **владельцу** — рекомендации по цене для быстрой продажи и стоит ли торопиться
  с учётом расходов на хранение и срока на стоянке;
- фильтр по **географии** (регион), бренду, вердикту и статусу.

## Как это работает

1. `ingest.py` скачивает таблицу как CSV, нормализует (регионы, бренды с
   кириллическими омоглифами, цены, годы), сохраняет снапшот и фиксирует
   изменения по VIN (новые / снятые / изменение цены) — всё одной транзакцией.
2. `valuation.py` оценивает машину через **Polza.ai** (OpenAI-совместимый прокси
   к Claude, `anthropic/claude-sonnet-5`, как в проектах Альфы). Оценка считается
   **один раз — при первом появлении машины в стоке**; пересчёт — только по
   команде (кнопка на карточке / «оценить необсчитанные»).
3. `scoring.py` — прозрачные формулы вердикта покупателю и рекомендаций продавцу
   поверх рыночной оценки и фактов из карточки.
4. Фоновый планировщик опрашивает таблицу **каждый час** (`POLL_INTERVAL_HOURS`)
   и авто-оценивает только вновь появившиеся машины.

## Хранилище

Данные пишутся в **общую базу проектов Альфы** (Postgres `alfa_collection`) в
**отдельную схему `stok_analiz`** — не пересекается с данными других приложений.
Если реквизиты Postgres не заданы, используется локальный SQLite (для отладки).

## Локальный запуск

```bash
export DB_HOST=... DB_NAME=... DB_USER=... DB_PASSWORD=... DB_PORT=5432   # Postgres Альфы
export DB_SCHEMA=stok_analiz
export POLZA_AI_API_KEY=...        # для оценки рынка
./run.sh                            # http://127.0.0.1:5060
```

Без Postgres-реквизитов приложение падает на SQLite-файл. Без `POLZA_AI_API_KEY`
работает всё, кроме оценки (кнопка вернёт «оценка не настроена»).

## Деплой на Render

`render.yaml` — Blueprint веб-сервиса. В Environment задать `DB_HOST`, `DB_NAME`,
`DB_USER`, `DB_PASSWORD` (те же, что у проектов Альфы), `DB_SCHEMA=stok_analiz`,
`POLZA_AI_API_KEY`. Схема и таблицы создаются автоматически при старте.

## Настройки (env)

| Переменная | Смысл | Дефолт |
|---|---|---|
| `DB_HOST/DB_NAME/DB_USER/DB_PASSWORD/DB_PORT` | Postgres Альфы | — |
| `DB_SCHEMA` | схема этого приложения | `stok_analiz` |
| `POLZA_AI_API_KEY` | ключ Polza.ai для оценки | — |
| `POLZA_MODEL` | модель | `anthropic/claude-sonnet-5` |
| `SHEET_ID` | ID гугл-таблицы | текущая |
| `STORAGE_COST_PER_DAY` | стоимость хранения, ₽/сутки | `500` |
| `POLL_INTERVAL_HOURS` | период автоопроса | `1` |
| `AUTO_VALUATE` | авто-оценка новых машин | `1` |
| `ADMIN_TOKEN` | пароль для изменяющих действий | пусто |
